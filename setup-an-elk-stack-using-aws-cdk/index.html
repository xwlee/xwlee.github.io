Setup an ELK stack using AWS CDK

13.01.2022 — aws, awscdk, typescript, elasticsearch — 1 min read

Prerequisites

Make sure you have installed Node.js 10.13.0 or later.

Make sure you have configured your workstation with your aws_access_key_id & aws_secret_access_key. If you have the AWS CLI installed, run the following command.

1# Provide your AWS access key ID, secret access key, and default region when prompted.
2aws configure

Make sure you have installed the AWS CDK Toolkit globally.

1npm install -g aws-cdk
2cdk --version

Initial Setup

In this tutorial, we will use TypeScript for the demo.

Let's initialize an empty AWS CDK project.

1mkdir cdk-elk && cd $_
2cdk init app --language typescript

Run the following command to verify that everything works correctly.

1npm run cdk synth

This will output an empty CloudFormation template inside cdk.out folder. You should see a file named CdkElkStack.template.json created.

Create an AWS Cognito User Pool

Add an import statement at the beginning of lib/cdk-elk-stack.ts

1import { Stack, StackProps } from "aws-cdk-lib";
2import { Construct } from "constructs";
3import * as cognito from "aws-cdk-lib/aws-cognito";
4

5export class CdkElkStack extends Stack {
6  constructor(scope: Construct, id: string, props?: StackProps) {
7    super(scope, id, props);
8

9    const userPool = new cognito.UserPool(this, "UserPool", {
10      userPoolName: `${Stack.of(this).stackName}UserPool`,
11      autoVerify: {
12        email: true,
13      },
14      standardAttributes: {
15        email: {
16          mutable: true,
17          required: true,
18        },
19      },
20    });
21    userPool.addDomain("UserPoolDomain", {
22      cognitoDomain: {
23        domainPrefix: "cdk-elk",
24      },
25    });
26  }
27}

A user pool is a user directory that stores your users that use to login to Kibana. We also need to specify the domain prefix for the Cognito login URL.

Create an AWS Cognito Identity Pool

Note: Here we omitted the code to create user pool

1import { Stack, StackProps } from "aws-cdk-lib";
2import { Construct } from "constructs";
3import * as cognito from "aws-cdk-lib/aws-cognito";
4

5export class CdkElkStack extends Stack {
6  constructor(scope: Construct, id: string, props?: StackProps) {
7    super(scope, id, props);
8

9    // User pool ...
10

11    const identityPool = new cognito.CfnIdentityPool(this, "IdentityPool", {
12      identityPoolName: `${Stack.of(this).stackName}IdentityPool`,
13      allowUnauthenticatedIdentities: false,
14    });
15

16    // See https://docs.aws.amazon.com/cognito/latest/developerguide/role-based-access-control.html
17    const authenticatedRole = new iam.Role(this, "AuthenticatedRole", {
18      assumedBy: new iam.FederatedPrincipal(
19        "cognito-identity.amazonaws.com",
20        {
21          StringEquals: {
22            "cognito-identity.amazonaws.com:aud": identityPool.ref,
23          },
24          "ForAnyValue:StringLike": {
25            "cognito-identity.amazonaws.com:amr": "authenticated",
26          },
27        },
28        "sts:AssumeRoleWithWebIdentity"
29      ),
30    });
31

32    new cognito.CfnIdentityPoolRoleAttachment(
33      this,
34      "IdentityPoolRoleAttachment",
35      {
36        identityPoolId: identityPool.ref,
37        roles: {
38          authenticated: authenticatedRole.roleArn,
39        },
40      }
41    );
42  }
43}

A few things to notice:

  • Identity pool creates unique identities for the authenticated user and attaches a role to that user.
  • The role defines the permissions to access the AWS resources.
  • Both user pool & identity pool will be used when we define the ES domain.

Create an ES domain

Note: Here we omitted the code to create user pool & identity pool

1import { CfnOutput, Stack, StackProps } from "aws-cdk-lib";
2import { Construct } from "constructs";
3import * as cognito from "aws-cdk-lib/aws-cognito";
4import * as iam from "aws-cdk-lib/aws-iam";
5import * as es from "aws-cdk-lib/aws-elasticsearch";
6import { EbsDeviceVolumeType } from "aws-cdk-lib/aws-ec2";
7import { ArnPrincipal, Effect } from "aws-cdk-lib/aws-iam";
8

9export class CdkElkStack extends Stack {
10  constructor(scope: Construct, id: string, props?: StackProps) {
11    super(scope, id, props);
12

13    // User pool ...
14

15    // Identity pool ...
16

17    // Allow Amazon ES to access Cognito
18    const esRole = new iam.Role(this, "EsRole", {
19      assumedBy: new iam.ServicePrincipal("es.amazonaws.com"),
20    });
21    esRole.addManagedPolicy({
22      managedPolicyArn: "arn:aws:iam::aws:policy/AmazonESCognitoAccess",
23    });
24

25    const domain = new es.Domain(this, "Elasticsearch", {
26      domainName: "cdk-elk",
27      version: es.ElasticsearchVersion.V7_10,
28      enableVersionUpgrade: true, // This allow in-place Elasticsearch version upgrade
29      capacity: {
30        dataNodeInstanceType: "t3.small.elasticsearch",
31        dataNodes: 1, // For testing purpose, we only create 1 instance
32      },
33      ebs: {
34        // Attach a 30GB GP2 volume
35        enabled: true,
36        volumeSize: 30,
37        volumeType: EbsDeviceVolumeType.GP2,
38      },
39      accessPolicies: [
40        // Allow authenticated users to access Kibana
41        new iam.PolicyStatement({
42          effect: Effect.ALLOW,
43          principals: [new ArnPrincipal(authenticatedRole.roleArn)],
44          actions: ["es:ESHttp*"],
45          resources: [
46            `arn:aws:es:${Stack.of(this).region}:${
47              Stack.of(this).account
48            }:domain/cdk-elk/*`,
49          ],
50        }),
51      ],
52      cognitoKibanaAuth: {
53        userPoolId: userPool.userPoolId,
54        identityPoolId: identityPool.ref,
55        role: esRole,
56      },
57    });
58

59    new CfnOutput(this, "ESKibanaUrl", {
60      description: "Elastic Search Kibana Endpoint.",
61      value: `https://${domain.domainEndpoint}/_plugin/kibana/`,
62    });
63  }
64}

What's going on here?

  • We declared an IAM role that allows Amazon ES to configure Cognito authentication for Kibana.
  • We defined an ES domain with the access policies that allow the authenticated role we created previously to access it.
  • We output the Kibana URL after the domain is created.

Create an User in User Pool to access Kibana

Go to AWS Cognito console to create a user to signin to Kibna. Click the Create User button in the Users tab of the Users and groups tab.

Fill in the Username, Temporary password, and Email. Check Mark email as verified? and hit the Create User button to create the user.

Go to the link display after the stack is created successfully to login to Kibana. When you log in for the first time, you will be prompted to change the temporary password.

Once you have reset the password, log in with the new password.

You should see this page after you log in successfully.
